## 1. 项目概述

本项目由两大系统组成：

### 1.1 AI资料库系统 (Phase 1)

基于RAG（检索增强生成）的儿童健康智能问答系统。核心能力：
- **多格式文档加载**: PDF/DOCX/TXT/MD/CSV/XLSX → 语义分块 → 向量化 → Milvus存储
- **智能问答**: 用户问题 → 查询重写 → 向量检索 → LLM重排序 → 上下文构建 → LLM生成 → SSE流式响应
- **症状分析**: 自动识别症状 → 风险分级（低/中/高/紧急）→ 建议类型分类
- **多租户**: 每个客户独立AI对话空间

### 1.2 多用户小程序工厂平台 (Phase 2)

SaaS平台，客户注册后可在线配置并一键生成微信小程序源码。核心能力：
- **8大功能模块**: 内容管理、AI健康顾问、电商商城、活动运营、预约服务、会员体系、数据分析、基础框架
- **可视化配置**: 品牌Logo、主题色、底部版权信息实时预览
- **AI增强**: 文章生成、活动文案、产品卖点提炼、活动复盘
- **一键构建**: 选择模块 → Node.js代码引擎编排 → ZIP源码包下载
- **全托管BaaS**: 生成的小程序直接调用BaaS API（内容/电商/活动/用户/预约），无需自建后端

```json
{
  "message": "宝宝消化不好怎么办？",
  "conversation_id": "uuid",
  "child_id": "1",
  "child_age_months": 6,
  "history": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
```

**SSE事件流：**

```
event: thinking  → data: {"stage":"retrieving", "message":"正在检索相关知识..."}
event: token     → data: {"content":"宝宝消化"}
event: token     → data: {"content":"不良可能"}
event: done      → data: {"message_id":"...", "content":"...", "assessment":{...}, "recommendations":[...], "sources":[...]}
```

## 5. 核心业务链路

### 5.1 客户创建小程序全链路

```
客户注册/登录 → 创建项目(填写信息+选模块+配置品牌)
  → 一键构建 → Go创建BuildTask(pending) → 异步调用CodeGen
  → Node.js: 模块解析(resolver) → 拓扑排序 → 模板拼装(compose) → 变量注入(injector) → ZIP打包(packager)
  → Go更新BuildTask(done+zip_url+md5) → 客户下载ZIP
  → 解压 → npm install → 微信开发者工具导入 → 发布上线
```

### 5.2 AI对话全链路

```
用户问题 → Go Biz透传 → Python AI
  → 查询重写(rewrite_query) → Milvus向量检索(retrieve) → LLM重排序(rerank)
  → 上下文构建(build_context) → System Prompt注入 → LLM流式生成(generate_stream)
  → SSE透传 ← Go Biz ← 客户端
```

### 5.3 BaaS数据流

```
小程序(UniApp) → POST /baas/v1/articles (Header: X-API-Key: ak_xxx)
  → Go BaaSAuth中间件(查MiniappProject表校验API Key)
  → ProjectIsolate中间件(注入project_id)
  → Handler(project_id过滤查询)
  → PostgreSQL
  → 返回该项目数据
```

## 7. UniApp模板体系

### 7.1 模板引擎流程

```
module.json (模块声明)
    ↓
resolver.js (依赖解析 → 拓扑排序 → 自动补齐)
    ↓
compose.js (复制base骨架 → 合并pages.json路由 → 拼装模块页面 → 生成README)
    ↓
injector.js (变量替换: {{appName}} {{primaryColor}} {{apiKey}} ...)
    ↓
packager.js (ZIP压缩 → MD5校验)
    ↓
输出: UniApp完整工程源码包
```

## 10. 部署指南

### 10.1 前置要求

| 软件 | 最低版本 | 说明 |
|------|----------|------|
| Docker | 24.0+ | 容器引擎 |
| Docker Compose | 2.x | 容器编排 |
| 内存 | 16GB+ | Milvus需要较大内存 |
| 磁盘 | 50GB+ | 向量数据+数据库+镜像 |

### 10.2 快速部署（Docker Compose）

```bash
# 1. 进入项目目录
cd hxbaby

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，至少配置：
#   JWT_SECRET=随机64位字符串
#   DASHSCOPE_API_KEY=你的通义千问API密钥
#   DB_PASSWORD=数据库密码

# 3. 一键部署
bash scripts/deploy.sh

# 4. 验证健康状态
bash scripts/health-check.sh
```

### 10.3 分步部署

```bash
# 构建镜像（可并行）
docker compose build --parallel

# 启动基础设施（PostgreSQL + Redis + MinIO + etcd + Milvus）
docker compose up -d postgres redis minio etcd milvus

# 等待基础设施就绪
sleep 15

# 启动AI服务
docker compose up -d ai-service

# 启动业务服务和代码生成
docker compose up -d biz-service codegen-service

# 构建前端并启动Nginx
cd factory-admin && npm install && npm run build && cd ..
docker compose up -d nginx

# 查看状态
docker compose ps
```

### 10.4 开发环境部署

```bash
# 启动基础设施
docker compose up -d postgres redis minio etcd milvus

# 各服务本地启动：
# Python AI:
cd ai-service && pip install -r requirements.txt && uvicorn main:app --port 8001 --reload

# Go Biz:
cd biz-service && go run cmd/main.go

# Node.js CodeGen:
cd codegen-service && npm install && npm start

# React Admin:
cd factory-admin && npm install && npm run dev
```

### 10.5 服务地址

| 服务 | 生产地址 | 开发地址 |

### 10.6 常用运维命令

